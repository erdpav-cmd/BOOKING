# Система бронирования ресторана

Десктоп-приложение на **Python** и **Tkinter** с хранением данных в **PostgreSQL**. Управление пользователями, столами и бронированиями с проверкой пересечения интервалов.

## Требования

- Python 3.10+
- Установленный и запущенный PostgreSQL

## Установка

1. Клонируйте или скопируйте проект, перейдите в каталог проекта.

2. Создайте виртуальное окружение (рекомендуется):

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Создайте базу данных в PostgreSQL (например, `booking`) и пользователя с правами на неё.

5. Скопируйте пример переменных окружения и укажите свои значения:

```bash
copy .env.example .env
```

Отредактируйте `.env`: `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`.

## Инициализация таблиц

Таблицы создаются из dataclass-моделей в каталоге `models/`:

```bash
python backend.py
```

Или из кода: `backend.create_tables()`.

## Запуск приложения

```bash
python app.py
```

## Структура проекта

| Файл / каталог | Назначение |
|----------------|------------|
| `app.py` | Графический интерфейс (вкладки: пользователи, столы, бронирования) |
| `backend.py` | Подключение к БД, создание таблиц, CRUD и проверка занятости стола |
| `postgres_driver.py` | Обертка над `psycopg2` (подключение, CRUD, генерация `CREATE TABLE` из моделей) |
| `models/` | Модели `User`, `Table`, `Booking` |
| `.env.example` | Шаблон переменных для подключения к PostgreSQL |

## Примечания

- Пароли в форме сохраняются как введённые строки (без хеширования); для продакшена нужна отдельная схема аутентификации.
- При смене схемы моделей для уже существующих таблиц может потребоваться миграция вручную в PostgreSQL.
